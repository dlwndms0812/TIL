### 서버의 특징

- 서버:클라이언트=1:N 관계
- 고정적인 개념이 아닌 상대적인 역할에 따른 관계

### 서버의 구성

- 클라이언트(web, 안드로이드, ios, 게임)
- 서버(Apache, Ngnix)
- 백엔드 언어(PHP, Spring, Node.js)
- DB, DBMS(MySQL, Oracle, Mongo db)
- 클라이언트의 요청을 받음→백엔드 언어→데베에 쿼리문 요청, 서버에게 요청을 가공 후 전달→서버는 클라이언트에게 전달

### APM

- Apache + Php + Mysql
